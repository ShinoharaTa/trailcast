# Trailcast

旅の記録やオフ会の記録を、スレッド形式でリアルタイムに残して共有するアプリ。

データは AT Protocol の Personal Data Server (PDS) に保存され、ユーザー自身がデータの所有権を持ちます。
Bluesky の標準タイムラインとは完全に分離された独自の Lexicon (`net.shino3.trailcast.*`) を使用し、既存の SNS 体験を汚染しません。

## コンセプト

**「今すぐシェアしつつ、記録として残す」**

旅行中やイベント中に、写真と短いテキストをチェックポイントとして投稿し、時系列のスレッドを作ります。
スレッドは後からでも編集・整理でき、OGP 対応の URL で誰にでも共有できます。

## 主な機能

- **スレッド（イベント）管理** -- タイトル・概要付きのスレッドを作成し、時系列でチェックポイントを投稿する
- **チェックポイント投稿** -- 写真（最大 4 枚）＋テキスト（200 文字）＋位置情報を 1 投稿として記録
- **公開設定** -- Private（自分のみ投稿可）/ Public（参加者全員が投稿可）
- **位置情報・メタデータ管理** -- 写真の EXIF（撮影位置・時刻）を保持し、あとから個別に削除・編集可能
- **ブックマーク** -- 他のユーザーの PDS にあるスレッドをお気に入り登録
- **Bluesky クロスポスト**（任意） -- スレッドの開始・ハイライトなどを Bluesky へリンク付きで投稿
- **OGP 対応の共有リンク** -- `trailcast.shino3.net` 上で、SNS カードとして表示できる URL を生成

## 非対応

- グローバルなスレッド一覧・検索（大規模な Relay / AppView が必要になるため）
- 投稿へのリプライ
- 投稿・スレッドへのリアクション（いいね・リポスト）

## 技術スタック（想定）

| レイヤー | 技術 | 備考 |
|---|---|---|
| アプリシェル | Tauri | デスクトップ / モバイルのマルチプラットフォーム |
| フロントエンド | Next.js (React) | Tauri の WebView 内で動作 |
| データ保存 | AT Protocol PDS | ユーザー自身の PDS にカスタム Lexicon で保存 |
| バックエンド（軽量） | Hono + Cloudflare Workers / Vercel | OGP 生成、Public Event の集約インデクサー |
