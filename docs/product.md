# プロダクト要件

## 目的

Trailcast は「今すぐシェアしつつ、記録として残す」ための旅・イベント記録アプリです。

旅行やオフ会のさなかに、写真とテキストをチェックポイントとして時系列に投稿し、1 つのスレッドにまとめます。
記録はリアルタイムに OGP 対応の URL で共有でき、あとからチェックポイントの時刻や位置情報を編集・整理できます。

## ユースケース

### UC-1: ソロ旅行の記録（Private Event）

1. ユーザーは「京都日帰り旅」というスレッドを Private で作成する
2. 各スポットで写真とメモをチェックポイントとして投稿する
3. 投稿には現在地の位置情報が自動で付与される（任意）
4. 帰宅後、チェックポイントの時刻や並び順を微調整する
5. 完成したスレッドの URL を SNS やメッセンジャーで共有する

### UC-2: オフ会の共同記録（Public Event）

1. 幹事が「技術書典アフター」というスレッドを Public で作成する
2. 参加者全員がそのスレッドにチェックポイントを投稿できる
3. 各参加者は自分の PDS に投稿データを保存するが、スレッド上では時系列に統合表示される
4. 幹事は任意のタイミングで Bluesky にハイライト付きのクロスポストを行う

### UC-3: 過去の旅を振り返る

1. ユーザーは自分のスレッド一覧から過去の記録を選ぶ
2. チェックポイントを時系列で辿りながら、写真・テキスト・位置情報を確認する
3. 不要になった位置情報や EXIF データを個別に削除する

### UC-4: 過去の Bluesky 投稿から旅を再構成する

1. ユーザーは新しいスレッドを作成し、「Bluesky 投稿からインポート」を選択する
2. 自分の過去の Bluesky 投稿（`app.bsky.feed.post`）が一覧表示される
3. チェックポイントにしたい投稿にチェックを入れて取り込む
4. テキスト・画像が Trailcast のチェックポイントとして生成される（チェックポイント時刻は元投稿の `createdAt`）
5. インポート元の Bluesky 投稿への参照リンクがチェックポイントに付与される
6. 位置情報は付与されない（過去の投稿のため現在地は無意味、Bluesky 上の画像は EXIF 削除済み）

### UC-5: 他人の旅をブックマークする

1. 共有リンクから他のユーザーのスレッドを閲覧する
2. 気に入ったスレッドをブックマーク（お気に入り）に追加する
3. ブックマーク一覧から、いつでもそのスレッドに再アクセスできる

## 機能要件

### スレッド（イベント）管理

| 項目 | 仕様 |
|---|---|
| 作成 | タイトル（必須）、概要（任意）を設定してスレッドを作成 |
| 公開設定 | Private（自分のみ投稿可）/ Public（参加者全員が投稿可） |
| 編集 | タイトル・概要はあとから編集可能 |
| 削除 | スレッドごと削除可能（紐づくチェックポイントも連動） |

### チェックポイント投稿

| 項目 | 仕様 |
|---|---|
| テキスト | 最大 200 文字 |
| 画像 | 最大 4 枚 / 投稿 |
| 位置情報 | 投稿時に位置情報トグルが ON ならデバイスの現在地（GPS）を記録する。OFF なら位置情報なし。あとから削除可能 |
| チェックポイント時刻 | 投稿時刻で初期記録し、あとから編集可能 |
| 写真メタデータ | EXIF（撮影位置・時刻）の読み取りはユーザーの任意（オプトイン）。保持した場合も意図的に削除可能。EXIF の位置情報はチェックポイントの `location` とは独立して管理する |
| Bluesky 投稿からのインポート | 自分の過去の Bluesky 投稿を選択してチェックポイントとして取り込める。インポート元の投稿 AT URI を参照として保持する。位置情報は付与されない |

### クロスポスト（Bluesky 連携）

| 項目 | 仕様 |
|---|---|
| 対象 | 任意のタイミングでユーザーが手動実行 |
| 内容 | テキスト ＋ Trailcast のスレッド URL をリンクとして含む `app.bsky.feed.post` |
| 用途例 | 「旅行始めました！」「ハイライトまとめ」などの通知的な投稿 |

### ブックマーク

| 項目 | 仕様 |
|---|---|
| 対象 | 他のユーザーの PDS にあるスレッド |
| 保存先 | 自分の PDS（カスタム Lexicon） |
| 操作 | 追加・削除 |

### 共有（OGP）

| 項目 | 仕様 |
|---|---|
| URL | `trailcast.shino3.net` 上のパーマリンク |
| OGP | スレッドのタイトル・概要・サムネイル画像をメタタグとして動的生成 |

## 対応プラットフォーム

| プラットフォーム | 方式 | 優先度 |
|---|---|---|
| デスクトップ（macOS / Windows / Linux） | Tauri アプリ | 高 |
| モバイル（iOS / Android） | Tauri Mobile | 検討中 |
| Web ブラウザ | `trailcast.shino3.net` での閲覧（OGP / 共有リンク先） | 高 |

- Tauri アプリとして PC・スマホの両方に配信することを目指す
- Web は最低限、共有リンク先としての閲覧体験を提供する（OGP 含む）
- アプリ配信（デスクトップインストーラー / モバイルストア）も将来的に検討

## 非対応範囲

- グローバルなスレッド一覧・検索（大規模 Relay / AppView が必要）
- 投稿へのリプライ
- 投稿・スレッドへのリアクション（いいね・リポスト）
